Introduction

In this project we aim to evaluate the propagation of knowledge through a distributed system, where nodes (or processors) are sparsely connected to each other. For instance, new knowledge obtained by node A which can be a sensor reading needs to be transferred throughout the network so all the nodes know about it, which can be done with public announcement logic. Additionally, knowledge can also be updated so facts that were valid at t are not valid at t+1. Topics to investigate:
  • Knowledge propagation.
  • Updating information.
  • Conflict resolution.
  • Changes in network topology.
Our research question is: Can the knowledge contained in a dynamically changing decentralized communications network be accurately modelled using Kripke models? The model would need to accurately model the various worlds created by the network structure and the traffic routing. The various nodes in the network would be the agents in our model, the routing of traffic would govern their knowledge and belief about the state of the network. We would look to include both public announcements and common knowledge into our model. We aim to implement as part of our website a small simulation of a network evolving and the knowledge of the nodes, as well as the Kripke model representing this simulated network. The way the model will evolve will cover the information propagating through the network, changes to the network including new nodes being added, nodes removed and temporary breaks in connections resulting in routing changes. We could consider adding deletion errors if it does not add too much complexity.
